# 小红书

查询数据来源接入小红书。

## 调研结论（2026-06-24）

**结论：当前不接入，原因如下，记录备用。**

### 官方 API 现状

小红书有官方开放平台（`school.xiaohongshu.com/en/open`），但定位是**企业商务合作**（商品 SPU/SPL 管理、品牌内容分发、营销投放），需要走商家资质审核，不对个人开发者开放，更没有通用内容搜索接口。

内容搜索（"上海徒步推荐"、"好吃的火锅"之类）在官方侧根本没有 API——这是小红书刻意保护的核心资产。

### 第三方方案

市面上的第三方"小红书API"（Rnote、justoneapi 等）均基于逆向工程 + 爬取，本质是伪造客户端请求或维护登录态。主要风险：

- **合规**：违反小红书 ToS 和数据版权，CNNIC《互联网信息服务深度合成管理规定》也有数据获取合规要求
- **稳定性**：小红书会定期更新签名算法、设备指纹，这类工具频繁失效（维护成本高）
- **封号**：账号/IP 有被封风险

### 替代方案

群里问"XX地方怎么玩"、"推荐餐厅"之类，DeepSeek 训练数据里已包含大量小红书风格内容，能给出合理建议。如果未来确实需要实时内容，更可持续的路径是接入一个通用**搜索 API**（如必应搜索 API），可同时覆盖小红书、大众点评、B站等，比单独维护一个小红书爬虫划算得多。

### 待办

- [ ] 待官方开放通用搜索 API 时重新评估
- [ ] 如果需要实时外部信息，考虑接入通用 Web 搜索工具（必应/Brave Search API）作为统一入口
