# 活动生命周期与权限

活动的增 / 删 / 改 / 参加，必须走二次确认 + 权限校验。

## 待办

- [ ] **新增**：用户描述意图后，bot 把解析出来的活动结构化展示，等用户回 "确认 / yes / 是" 才落库。
- [ ] **删除**：同样要二次确认；只允许发起者或管理员执行。
- [ ] **修改**：仅发起者可改；他人改请求要被拒绝并给出说明。
- [ ] **参加**：bot 不直接把人加进活动，而是引导参加者私聊或群里 @ 发起者。
- [ ] 状态机用代码实现，不要塞给 LLM；LLM 负责语义识别意图，代码负责状态流转。

## 注意点

- 二次确认期间要带过期时间，避免悬挂。
- 状态变更要落库，方便排错。
- 错误统一走 `errorStore.logError('event', ...)`。
