# `/统计 群` `/分析 群` 仅当前群

## 背景

`src/platforms/wechat/commandRouter.js` 的 `/help` 菜单里向所有用户暴露了：

```
/统计 群 <群名>
/分析 群 <群名>
/分析 好友 <备注>
```

当前实现里 `<群名>` 是用户传什么就查什么，没校验调用者是不是该群成员，违反「群隔离」红线 —— A 群成员理论上能拉到 B 群的统计 / 分析。

## 现状

- 入口：`commandRouter.js` 里 `if (['分析', 'analyze', '统计', 'stats'].includes(command))` 那一段。
- `parseTarget(tokens)` 直接把 `tokens.slice(2)` 当群名传给 `analyzeWechatMessages`。
- `context.roomName` 已经传进 `handleAdminCommand` 了（`/help`、`/活动` 都在用），但分析分支没用上。

## 待办

- [ ] 群聊里调用 `/统计 群` `/分析 群` 时，无论用户传不传 `<群名>`，强制用 `context.roomName` 覆盖；用户传了别的群名直接拒绝并给出说明。
- [ ] 私聊里调用这两条指令，要么禁掉，要么走管理员白名单（参考 `getWechatRuntimeConfig` 现有的 admin / 白名单约定，没有的话先按「私聊禁用」最稳）。
- [ ] `/分析 好友 <备注>` 同样要加权限：默认只有管理员能查任意人；普通用户最多查自己。
- [ ] 改完在 `/help` 文案里把 `<群名>` 参数去掉，避免误导（群里直接 `/统计 群` 就够了）。
- [ ] `node --check` 过一遍，再写最小验证：模拟群里调用、私聊调用、跨群调用三种路径。

## 注意点

- 按 project-guidelines，「权限 / 隐私边界」必须代码兜底，不能塞给 LLM。
- 错误统一走 `errorStore.logError('command', err, { command, roomName })`。
- 别忘了 `analyzeWechatMessages` 里 `getServe('ChatGPT')` 这条路径 —— 如果当前部署没配 OpenAI key，`/分析` 会直接抛错，这里也顺手判断下并给友好提示。

## 完成记录

- 2026-06-24：`commandRouter.js` 的 `/分析 /统计` 分支加了权限边界（代码兜底）：群分析强制锁定 `context.roomName`，用户传别的群名直接拒绝；`/统计 /分析 好友` 默认只能查自己，传别人拒绝；私聊禁用群/好友分析；`analyzeWechatMessages` 调用包了 try/catch，缺模型 key 时给友好提示并 `logError('command', ...)`。`/help` 菜单去掉了 `<群名>` 参数。已写最小验证覆盖：群内默认本群、跨群拒绝、无参默认、查他人拒绝、查自己、私聊禁用。
