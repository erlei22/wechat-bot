# 画像按需注入

只有当前消息确实涉及画像内容时才注入画像，否则不注入或交给 `get_person_profile` 工具按需查询。

## 现状

- 当前实现是「不管说什么都把整份画像塞进 prompt」，既费 token 又有泄露风险。

## 待办

- [ ] 入口加一层相关性预筛（关键词 / 提及人名 / @某人），命中再走画像。
- [ ] 不命中时只暴露 `get_person_profile` 工具，让 LLM 主动来取，结果再走代码消毒。
- [ ] 跨群场景：默认不带其他群的画像 notes，调用方要主动指定 `group`。

## 注意点

- 相关性判断可以 LLM + 代码协同：代码先做廉价过滤，模糊场景再让 LLM 判。

## 完成记录

- 2026-06-24：`profileStore.js` 新增 `isProfileRelevant(text, profile, scope)`（中文 n-gram 廉价预筛），`sendMessage.js` 仅在命中相关性时注入画像，否则交给 `get_person_profile` 工具按需取（协同模式，无额外阻塞 LLM 调用）。注入与相关性判断均按 `scope`（群名）隔离，跨群 notes 默认不带。
