# 区分性别

画像增加性别字段。

## 待办

- [ ] `profileStore.js` 的 schema 加 `gender` 字段（值：`male` / `female` / `unknown` / 自定义）。
- [ ] 抽取规则：从消息里识别（自我介绍、被叫"姐 / 哥 / 美女 / 帅哥"等），结果要 LLM 判 + 代码消毒后才写。
- [ ] 默认 `unknown`，不强行猜测。
- [ ] 注入 prompt 时，性别只在确认非 unknown 才带。

## 注意点

- 这是敏感字段，错了体验非常差，宁缺毋滥。
- 用户可以通过指令手动修正（这条要不要做单独评估）。

## 完成记录

- 2026-06-24：`profileStore.js` 新增 `sanitizeGender(raw, existing)`（只接受 `male`/`female`/`unknown`，已有确认值不被 `unknown` 覆盖）。两处 LLM 提取 prompt 加 `gender` 字段说明 + 宁缺勿滥规则；结果经消毒后写入 profile（初始化带 `gender: 'unknown'`）。`formatProfileForPrompt` 只在确认非 unknown 时注入性别。`commandRouter.js` 的 `/画像` 展示加性别行（`unknown` 显示 `—`）。最小验证：`male`/`female` 注入、`unknown`/无字段不注入，通过。
