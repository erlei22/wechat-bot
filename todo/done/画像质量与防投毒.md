# 画像质量提升 + 防投毒

## 背景

当前画像从「单条消息」提取，产出大量揣测（如小白画像里「对群主感情状况感兴趣但担心直接询问」这种瞎猜），质量差。且缺乏对恶意/脏数据的防御。

用户决策：
- 实时性先保持高（画像现在太空，要快点填起来）
- 隐私不考虑（本来就是群聊记录）
- 不做自助申报
- **重点：防止有人喂脏数据**

## 现状

- `extractFromPassiveMessage`：单条消息 + 已有画像 → LLM 增量提取，15 分钟/人冷却
- `extractAndUpdateProfile`：@bot 回复后同理
- 已有 manipulation 检测（粗）
- 代码消毒：tags/notes 长度、gender 枚举

## 方案

### 1. 滚动上下文（质量）
提取时不只喂单条消息，从 `messages.db` 拉该发言人最近 N 条消息（同群、近几天）作为上下文整段喂 LLM。实时触发不变，但模型看到的是对话而非孤立单句 → 揣测大幅减少。
- 冷却从 15 分钟降到 ~5 分钟，让画像填得快些
- 可记录每人「上次提取到的消息 ts」，只处理增量 + 少量前文

### 2. 防投毒四层
- **归属隔离（代码）**：一条消息只更新发言人自己的画像；提取结果如果主语是别的群成员（命中已知 name 列表），整条丢弃
- **提取 prompt（LLM）**：只采纳本人对自己的陈述；排除玩笑/调侃；排除对他人的评价；排除负面指控/侮辱；检出指令注入和越权
- **代码消毒（确定性兜底）**：
  - 结构化字段枚举/正则/长度校验（已有，扩展到新字段）
  - 侮辱/指控词黑名单（渣男、骗子、老赖…）→ 丢弃
  - 第三方姓名扫描：note/tag 含其他群成员名 → 丢弃
- **置信度 + 溯源**：每条软事实记 `count` + 最近来源；软印象需 ≥2 次观察才注入 prompt；硬事实（有车/城市/职业）单次可采纳但标 tentative

### 3. 管理与回溯
- `/画像 重置 <昵称>`：清空某人画像
- `/画像 删 <昵称> <字段>`：删某个被污染的字段
- 每条提取记来源 message，方便人工核查

## 注意点

- LLM 判语义、代码做兜底，两层都要有
- 改完旧画像兼容（缺字段 fallback）
- 玩笑识别别太严，否则啥都提取不到——朋友群本来就爱开玩笑
